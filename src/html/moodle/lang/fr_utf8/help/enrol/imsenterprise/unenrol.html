Désinscrire des participants

Les données IMS Enterprise peuvent ajouter, mais aussi retirer des inscriptions aux cours, pour tous les participants. Si ce réglage est activé, Moodle effectuera effectivement les désinscriptions indiquées dans les données.

Il y a trois façon de désinscrire les participants dans les données IMS entreprise :

  • un élément <member> qui indique le participant et le cours, et dont l'attribut « recstatus » a la valeur 3, pour l'élément <role> (ce qui signifie suppression). CECI N'EST PAS ENCORE IMPLÉMENTÉ DANS LA MÉTHODE D'INSCRIPTION.
  • un élément <member> qui indique le participant et le cours, et dont l'élément <status> a la valeur 0 (qui signifie inactif).

La troisième façon est légèrement différente. Elle ne requiert pas l'activation de la présente option, et peut être spécifiée à l'avance :

  • un élément <member> qui spécifie un élément <timeframe> pour l'inscription peut indiquer le début et/ou la fin de l'inscription d'un participant. Ces dates, si elles sont présentes, sont enregistrées dans la table des données d'inscription de Moodle, ce qui a pour conséquence de désinscrire automatiquement l'étudiant à la date d'échéance.
